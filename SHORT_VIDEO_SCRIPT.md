# 60-90 Second Video Script

Use this if a short video version is needed for follow-up promotion.

## Hook

Kaching is not just another crypto lottery. The interesting part is how it turns a World Cup attention cycle into a verifiable Solana draw loop.

## Main Explanation

The user path is simple:

Landing page -> Next Draw -> Buy Ticket -> Wait for Draw -> View Result -> View OnChain Proof -> Check My Tickets -> Refer or return for the next draw.

That flow matters because most users do not start by caring about infrastructure. They care about a clear game, a clear result, and a reason to come back.

Kaching's strongest surfaces are Next Draw, Buy Ticket, View Result, View OnChain, My Tickets Dashboard, My Referral, and Leaderboard. Together, they can turn each draw into a repeatable campaign moment.

Solana matters underneath because lottery-style products need fast, low-cost interactions and public verification. The chain is not decoration here; it supports the trust claim.

## Close

My three recommendations:

1. Make onboarding clearer with countdown, pot size, and ticket status.
2. Make proof understandable before sending users to an explorer.
3. Build the WC campaign around match-day moments, leaderboards, referrals, and repeat draws.

If Kaching makes the result clear and the proof one click away, it can turn on-chain verification into a consumer feature users actually understand.
